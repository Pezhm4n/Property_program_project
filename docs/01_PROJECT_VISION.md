# PROJECT VISION

## هدف پروژه (Project Goal)
ارتقای پروژه دانشگاهی «مدیریت بنگاه املاک» که پیش‌تر با زبان C و بر پایه فایل‌های متنی نوشته شده بود، به یک نرم‌افزار دسکتاپ صنعتی، امن و کارآمد. هدف این است که هسته اصلی C حفظ و Refactor شود و در قالب یک DLL، منطق کسب‌وکار و ارتباط با پایگاه داده (SQLite) را مدیریت کند، در حالی که رابط کاربری کاملاً مدرن با Python و PySide6 طراحی می‌شود.

## نیازمندی‌های غیرعملیاتی (Non-Functional Requirements - NFR)
- **Performance Target:** زمان پاسخ‌دهی به کوئری‌ها و جستجوهای پیچیده زیر 200 میلی‌ثانیه.
- **Memory Target:** مصرف حافظه هسته C (غیر از UI) زیر 100 مگابایت در حالت بارگذاری کامل.
- **Startup Time:** زمان بارگذاری اولیه نرم‌افزار زیر 2 ثانیه روی سیستم‌های استاندارد.
- **Maximum DB Size:** پشتیبانی بدون افت عملکرد تا حجم 10 گیگابایت برای پایگاه داده.
- **Maximum Property Count:** پشتیبانی تضمین شده از حداقل 1 میلیون رکورد ملک با ایندکس‌گذاری مناسب.
- **Maximum Concurrent Users:** پشتیبانی از دسترسی همزمان تا 5 کاربر روی شبکه محلی با استفاده از SQLite WAL Mode.
- **Backup Strategy:** بک‌آپ‌گیری روزانه، خودکار و بدون ایجاد وقفه (Non-blocking) با استفاده از Online Backup API در دیتابیس SQLite.
- **Recovery Strategy:** امکان بازیابی فوری پایگاه داده از طریق جایگزینی فایل `.db` بک‌آپ گرفته شده با یک کلیک توسط ادمین.

## محدوده پروژه (Scope)
- **کاربری:** نرم‌افزار دسکتاپ (Desktop Application) مختص سیستم‌عامل ویندوز.
- **مقیاس:** مناسب برای یک دفتر املاک منفرد (Single-Office) یا شبکه لوکال.
- **کاربران:** سیستم چندکاربره شامل مدیر (Admin) و مشاوران (Agents).
- **توزیع و استقرار:** توزیع پرتابل (Portable Distribution).
- **انواع املاک:** مسکونی (Residential)، اداری (Office)، و زمین (Land).
- **نوع معاملات:** فروش (Sale) و رهن/اجاره (Rent).
- **پایگاه داده:** استفاده از دیتابیس محلی (Embedded SQLite).
- **گزارش‌گیری:** ارائه گزارش‌های پیشرفته با خروجی PDF/Excel.

## خارج از محدوده (Out of Scope)
- سیستم‌های ابری (Cloud-based / SaaS) در این فاز.
- معماری Multi-Tenant (چند شعبه با داده‌های کاملاً مجزا در یک دیتابیس تحت وب).

## مخاطب (Target Audience)
آژانس‌های املاک کوچک تا متوسط که نیازمند نرم‌افزاری سریع، آفلاین و امن برای مدیریت داده‌های املاک و مشاورین خود هستند.

## ویژگی‌های اصلی (Key Features)
1. **امنیت داده‌ها (Secure Authentication):** استفاده از الگوریتم‌های استاندارد امنیتی برای رمزهای عبور و محافظت در برابر حملات Brute-Force.
2. **یکپارچگی تراکنش‌ها:** استفاده از SQLite برای جلوگیری از نقص داده.
3. **قابلیت حسابرسی کامل (Auditability):** Every critical business operation is auditable. (تمامی عملیات حساس، از لاگین تا بایگانی ملک، ثبت می‌شوند).
4. **رابط کاربری واکنش‌گرا و جذاب:** طراحی UI مدرن و زیبا با پشتیبانی کامل از زبان فارسی.

## چشم‌انداز آینده (Future Vision)
پروژه به عنوان یک **Platform** طراحی می‌شود. استفاده از ارتباط مبتنی بر JSON و تثبیت ABI در هسته C، این امکان را در آینده فراهم می‌کند که هسته فعلی (re_core.dll) از طریق ایجاد یک **REST API** به عنوان یک سرویس‌دهنده عمل کند. کلاینت‌های متعددی نظیر FastAPI، Electron، برنامه‌های Mobile، و CLI می‌توانند بدون کوچکترین تغییری در Business Logic از این هسته استفاده کنند.
