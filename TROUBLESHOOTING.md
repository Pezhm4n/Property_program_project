# راهنمای رفع مشکلات برنامه مدیریت املاک

این راهنما برای کمک به حل مشکلات رایج در اجرای برنامه مدیریت املاک تهیه شده است.

## خطای بارگیری کتابخانه C

### خطای `[WinError 193] %1 is not a valid Win32 application`

این خطا معمولاً به دلیل عدم سازگاری بین معماری کتابخانه C (32 بیتی یا 64 بیتی) و نسخه پایتون شما رخ می‌دهد.

**راه حل‌ها:**

1. **کامپایل مجدد کتابخانه C برای معماری مناسب:**
   - برای سیستم 64 بیتی: اجرای فایل `compile_lib_win64.bat`
   - برای سیستم 32 بیتی: اجرای فایل `compile_lib_win32.bat`

2. **اطمینان از نصب کامپایلر GCC:**
   - برای نصب MinGW-w64، می‌توانید از [MSYS2](https://www.msys2.org/) استفاده کنید.
   - پس از نصب MSYS2، دستور زیر را در ترمینال MSYS2 اجرا کنید:
     ```
     pacman -S mingw-w64-x86_64-gcc
     ```

3. **بررسی نسخه پایتون:**
   - در صورتی که از پایتون 64 بیتی استفاده می‌کنید، کتابخانه C نیز باید 64 بیتی باشد.
   - برای بررسی نسخه پایتون، دستور زیر را در ترمینال اجرا کنید:
     ```
     python -c "import platform; print(platform.architecture()[0])"
     ```

### خطای `cannot load library 'path\to\property_lib.dll'`

این خطا ممکن است به دلیل عدم وجود فایل DLL در مسیر مشخص شده یا عدم دسترسی به آن باشد.

**راه حل‌ها:**

1. **اطمینان از وجود فایل DLL:**
   - بررسی کنید که فایل `property_lib.dll` در مسیر `Property_program_project\lib\` وجود داشته باشد.
   - در صورت عدم وجود، فایل‌های batch کامپایل را اجرا کنید.

2. **کپی کردن DLL در مسیر سیستمی:**
   - فایل DLL را در مسیر `C:\Windows\System32` (برای سیستم 64 بیتی) یا `C:\Windows\SysWOW64` (برای برنامه‌های 32 بیتی در سیستم 64 بیتی) کپی کنید.
   - فایل DLL را در کنار فایل اجرایی پایتون (.exe) کپی کنید.

3. **اضافه کردن مسیر DLL به PATH سیستم:**
   - مسیر حاوی DLL را به متغیر محیطی PATH سیستم اضافه کنید.

## خطاهای مربوط به محیط برنامه

### خطای مربوط به کتابخانه‌های پایتون

اگر با خطای عدم وجود کتابخانه‌های پایتون مواجه شدید:

1. **نصب وابستگی‌ها:**
   ```
   pip install -r requirements.txt
   ```

2. **بررسی نسخه پایتون:**
   - این برنامه نیاز به پایتون 3.6 یا بالاتر دارد.

## خطاهای دسترسی به فایل‌ها

### خطای عدم دسترسی به فایل‌های داده

1. **اطمینان از وجود پوشه‌های مورد نیاز:**
   - اطمینان حاصل کنید که پوشه‌های `data`، `logs`، و `lib` وجود دارند.

2. **تنظیم مجوزهای فایل:**
   - اطمینان حاصل کنید که کاربر فعلی مجوز نوشتن در این پوشه‌ها را دارد.

## ارتباط با پشتیبانی

در صورت بروز مشکلات بیشتر، لطفاً با تیم پشتیبانی تماس بگیرید و اطلاعات زیر را ارائه دهید:
1. خطای دقیق
2. معماری سیستم شما (32 بیتی یا 64 بیتی)
3. نسخه پایتون
4. فایل‌های لاگ از پوشه `logs`
